A DFA is a finite-state machine that accepts or rejects a given string of symbols by running through a state sequence uniquely determined by the string.
A DFA is defined by a 5-tuple (Q, Σ, δ, q0, F), where:
- Q is a finite set of states
- Σ is a finite set of symbols called the alphabet
- q0 is the start state
- F is the set of accept states
- δ : Q × Σ → Q is the transition function

A Non-Deterministic Finite Automaton (NFA) is a finite-state machine that accepts or rejects a given string of symbols by running through a state sequence that may not be uniquely determined by the string.
An NFA is defined by a 5-tuple (Q, Σ, δ, q0, F), where:
- Q is a finite set of states
- Σ is a finite set of symbols called the alphabet
- q0is the start state
- F is the set of accept states
- δ : Q × Σ → P(Q) is the transition function.

The main difference between DFA and NFA is that in DFA, for each input symbol, there exists exactly one next state. In contrast, for an NFA, there may exist many paths for specific input from the current state to the next state.

PUMPING LEMMA

Assignment

10/10

The pumping lemma is a method used in the theory of formal languages to prove that a language is not regular. It states that for any regular language L, there exists an integer pumping length P such that any string s in L of length greater than or equal to P can be divided into three pieces, s = xyz, satisfying the following conditions:

- xyiz ∈ L, for i ≥ 0

- |y| > 0

- |xy| ≤ P

Context Free Grammar

This context free grammar is for L = {W ∈ {a,b}* | W is palindrome} S produces string that is equal in both sides, if there is "a" in left there is also "a" in right this ensures that the string is palindrome. And if S produces "a" or "b" the expression ends with a string equal both sides and "a" or "b" in the center.

Context-free grammars (CFGs) are used in compilers and in particular for parsing, taking a string-based program and figuring out what it means. CFGs are used to define the high-level structure of a programming language. Figuring out how a particular string was derived tells us about its structure and meaning.




Push-Down Automata

This PDA is for the language L = {W ∈ {0,1}* | W is palindrome}. It reads the first half of the string and push every element in a stack. The other half the element is popped, if the element matches the top of the stack in every pop and the remeining is an empty string, then the string is palindrome
Pushdown automata (PDAs) are used in many real-life applications. One example is whenever your smartphone calculator calculates an arithmetic expression, it uses an implementation of the pushdown automata to evaluate it. It verifies that the parentheses are balanced. In the absence of parentheses, it makes sure that the multiplication operator has more precedence than the addition operator. PDAs are also used in compilers to parse programming languages. They are used to recognize context-free languages. PDAs are used to recognize languages that cannot be recognized by finite automata.
